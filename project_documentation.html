Overview

What is AfricanNLP?

A full-stack research assistant that scrapes ArXiv for papers about African languages and NLP, stores them in a local ChromaDB vector database, and exposes an AI chatbot (ReAct Agent) that can search, summarize, and generate deep research reports.

Blog-Style Feed

Browse the latest AfricanNLP papers in a responsive card layout with category filters for 12+ African languages.

AI Chatbot

LangChain ReAct agent backed by Gemini, Groq, or OpenAI that searches the vector store and answers research questions.

Deep Research Agent

Autonomous pipeline: Search → Select → Read PDF → Generate structured report using Gemini 1.5 Flash.

Analytics Dashboard

Plotly-powered charts showing publication activity over time and topic/category distribution.

TL;DR Summaries

One-click Gemini-powered 3-bullet summaries of any paper abstract (problem → method → result).

Text-to-Speech

Listen to paper titles and abstracts via Google TTS (gTTS) — ideal for on-the-go learning.

Deep Dive

Module Reference

Detailed documentation of every module in the arxiv_bot package.

app.py — Application Entry Point

Orchestrator that creates the Gradio UI, wires all events, and launches the server. Initializes the Arxiver bot and GeminiProcessor at module level.

Function Description I/O
respond(message, history) Chat handler — passes user message to the ReAct agent str → str
add_paper_btn_click(paper_id) Adds a paper to the vector DB via the agent str → str
refresh_feed() Reads all papers/*.json, sorts by mtime, returns HTML → HTML
filter_feed(keyword) Filters feed cards by language/topic keyword str → HTML
summarize_paper(title) Generates TL;DR via Gemini (fallback: Groq agent) str → str
read_paper(title) Generates TTS audio of title + abstract via gTTS str → .mp3 path
run_scraper() Triggers bulk ArXiv fetch (150 papers) → str status
run_research(topic) Runs the autonomous deep research pipeline str → (status, report)
The app supports three LLM backends in priority order: Gemini → Groq (Llama 3) → OpenAI (GPT-3.5). Set the corresponding API key in .env.
manager/bots.py — Arxiver Agent

The brain of the system. Implements a LangChain ReAct agent with conversational memory and two custom tools.

Class Description
Arxiver Main agent. Initializes LLM, ChromaDB memory, conversational buffer, and ReAct agent with a custom prompt template.
AddArticleTool LangChain BaseTool — downloads a paper, chunks with tiktoken, upserts embeddings into ChromaDB.
GeminiLLM Custom LangChain LLM wrapper around google.generativeai with safety filters disabled.

Agent Tools

Tool Name Function
Search ArXiv Library Semantic search over ChromaDB via RetrievalQAWithSourcesChain
Add to ArXiv Library Downloads PDF → extracts text → chunks → embeds → stores in ChromaDB
knowledge_base/ — Vector Store & Paper Processing

constructors.py

Class Description
Arxiv Loads a paper by ID — downloads PDF, extracts text, fetches metadata via ArXiv API, splits into 300-token chunks (40 overlap) using tiktoken p50k_base.
ArxivGraphScraper Builds a citation graph from a seed paper. Recursively follows references across N levels.

database.py — LocalDatabase

Wraps ChromaDB with HuggingFace sentence-transformers/all-MiniLM-L6-v2 embeddings (384-dim). Batch add() with configurable batch size (default 64).

scraper.py — ArXiv Bulk Scraper
Feature Detail
Default Query cat:cs.CL AND ("African languages" OR "Hausa" OR "Yoruba" ...)
Keyword Chunking Reads keywords.txt (31 KB), splits into 15-term chunks to stay under ArXiv URL limits
Deduplication Skips papers already saved as JSON; strips version suffixes from IDs
Output Saves each paper as papers/{id}.json with title, summary, authors, categories, dates
agents/ — Deep Research Pipeline

researcher.py — ResearcherAgent

A linear 4-step autonomous agent:

  1. Search — Queries ArXiv for top 3 papers matching the topic
  2. Select — Uses Gemini to pick the most relevant paper ID
  3. Read — Downloads PDF, extracts first 5 pages of text
  4. Report — Generates a structured report (Executive Summary, Methodology, Findings, Relevance)

tools.py — ResearchTools

Method Description
search_arxiv(query, max_results) Returns formatted list: [ID] Title (Year) - PDF URL
read_paper(paper_id) Downloads PDF → extracts text from first 5 pages via pypdf
ai_processor.py — Gemini Summarizer

GeminiProcessor wraps google.generativeai (model: gemini-1.5-flash) for paper summarization.

Mode Prompt Strategy
tldr 3-bullet summary: Problem → Method → Impact
detailed Full paragraph summary of the abstract
gui.py / templates.py / logger.py

gui.py — Renders paper cards as HTML with hover effects, category pills, author badges, and PDF links. Provides the Gradio custom CSS theme.

templates.py — Reference extraction prompt: instructs LLM to parse raw citation text into structured Title | Authors | Year format.

logger.py — Color-coded console logging via colorlog (green = INFO, yellow = WARNING, red = ERROR).

Programmatic Usage

from arxiv_bot.manager.bots import Arxiver

bot = Arxiver(verbose=True)

# Ask a question
response = bot("What is attention?")
print(response['output'])

# Add a paper by ArXiv ID
bot("Add paper 1706.03762")

Scraper API

from arxiv_bot.scraper import scrape_arxiv

# Fetch 150 AfricanNLP papers
status = scrape_arxiv(
  query="AfricanNLP",
  max_results=150,
  save_dir="papers"
)

Deep Research API

from arxiv_bot.agents.researcher \
  import ResearcherAgent

agent = ResearcherAgent()
report = agent.run_deep_research(
  "Transformers for low-resource
   languages"
)

AI Summarization API

from arxiv_bot.ai_processor \
  import GeminiProcessor

gemini = GeminiProcessor()
summary = gemini.summarize(
  "Abstract text here...",
  type="tldr" # or "detailed"
)

Getting Started

Setup & Deployment

Local Development

# 1. Clone the repository
git clone https://github.com/alawalmuazu/AfricaNLP.git
cd AfricanNLP

# 2. Install dependencies
pip install -r requirements.txt

# 3. Set your API key
echo GEMINI_API_KEY=your_key > .env

# 4. Launch (default port 3000)
python app.py

Docker Deployment

# Build the image
docker build -t africannlp .

# Run the container
docker run -p 3000:3000 \
  -e GEMINI_API_KEY=your_key \
  africannlp
Uses python:3.10-slim with HF_HOME=/tmp/.huggingface for containerized model caching.

Environment Variables

Variable Required Description
GEMINI_API_KEY Primary Google Gemini API key for summarization and deep research
GROQ_API_KEY Optional Groq API key — uses Llama 3.1 8B Instant
OPENAI_API_KEY Optional OpenAI API key — fallback to GPT-3.5 Turbo
PORT Optional Server port (default: 3000)

Support

Troubleshooting

Common issues and their solutions.

"No API Key found" error on startup

The app requires at least one LLM API key. Create a .env file in the project root:

GEMINI_API_KEY=your_google_api_key_here

Alternatively, set GROQ_API_KEY or OPENAI_API_KEY.

ChromaDB permission or lock errors

ChromaDB writes to ./chroma_db/. If you see lock errors, ensure no other process is using the database. Delete the chroma_db/ directory and restart — it will be recreated automatically.

# Delete and restart
rm -rf chroma_db/
python app.py
Scraper returns 0 papers

This usually means ArXiv's API is rate-limiting or temporarily unavailable. Wait a few minutes and try again. The scraper has built-in retry logic but respects rate limits.

ArXiv limits API requests. Fetching 150 papers across multiple keyword batches may take 1–2 minutes.
Gemini API quota exceeded (429 error)

The free tier of Gemini has rate limits. The app handles this gracefully and returns an error message. Options:

  • Wait a few minutes and retry
  • Switch to Groq (free, fast) by setting GROQ_API_KEY
  • Upgrade to a paid Gemini API plan
Port already in use

If port 3000 is occupied, either kill the existing process or set a different port:

# Use a different port
PORT=8080 python app.py
sentence-transformers model download slow or fails

On first run, the app downloads all-MiniLM-L6-v2 (~80 MB). Ensure internet access. For Docker, the model is cached in /tmp/.huggingface. If behind a proxy, set HF_HOME and TRANSFORMERS_CACHE environment variables.
