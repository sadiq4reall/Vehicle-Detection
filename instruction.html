📚 Chapter One

Introduction

1.1 Background of the Study

Rapid urbanization, population growth, and the rising ownership of motor vehicles have placed tremendous pressure on urban transport systems, leading to persistent traffic congestion, longer travel times, and significant safety concerns. These challenges are particularly severe in developing regions where road infrastructure expansion has not kept pace with increased mobility demands (Wang et al., 2020).

Conventional traffic monitoring methods — including manual counting, inductive loop detectors, pneumatic tubes, infrared sensors, and radar — have been widely used for decades. However, these techniques are intrusive, expensive to maintain, limited in scalability, and often inaccurate under adverse environmental conditions (Zhao et al., 2017; Lv et al., 2015).

Recent advancements in computer vision and deep learning have transformed traffic surveillance by enabling automated, high-accuracy detection and classification directly from video streams. Deep learning–based models such as YOLO, Faster R-CNN, and SSD have demonstrated strong performance in extracting traffic information from CCTV footage (Redmon et al., 2016; Ren et al., 2015; Liu et al., 2016).

To address deployment limitations, the present study focuses on developing a lightweight but robust deep-learning-based system capable of detecting, classifying, and counting multiple categories of vehicles in real time using YOLOv11, benchmarked against Manoj et al. (2024) and Maurya et al. (2023).

1.2 State of the Art

Recent advancements in ITS have shifted from traditional sensing methods to camera-based, deep-learning solutions due to their non-intrusive, scalable, and cost-effective nature (Ma et al., 2015). The YOLO architecture has evolved through multiple versions, improving object localization, robustness to occlusion, and deployment on low-power devices. YOLOv11 further enhances these capabilities through lightweight modules and advanced feature-fusion.

1.3 Problem Statement

Efficient traffic management increasingly relies on real-time vehicle detection and classification systems. Region-based models such as Faster R-CNN are computationally intensive and unsuitable for time-critical applications. Single-stage detectors like YOLO still encounter difficulties in complex traffic environments involving occlusion, high vehicle density, and variations in object scale. There is a pressing need for a lightweight, robust, and real-time deep learning solution.

1.4 Aim and Objectives

Aim: To develop a Deep-learning Approach for Detection and Classification of Vehicles for Traffic Management.

Objective 1

Develop a traffic vehicle detection and classification model using YOLOv11 and Faster R-CNN deep learning architectures.

Objective 2

Simulate the proposed model using existing traffic datasets.

Objective 3

Assess and benchmark performance against existing traffic management algorithms.

1.5 Significance of the Study

This study contributes to effective traffic management by employing YOLOv11 for accurate, real-time vehicle detection and counting. It integrates a lightweight architecture with enhanced robustness under occlusion and high-density conditions, making it suitable for deployment on edge devices in resource-constrained environments.

1.6 Scope of the Study

This research focuses on detecting, classifying, and counting vehicles into four categories. Benchmarking is done against Faster R-CNN, earlier YOLO versions, and related studies. Validation is limited to public datasets and custom-annotated traffic video data.

⚙ i. Traffic Control Systems

Traffic control systems form the operational backbone — traffic lights, road signs, lane markings, and roundabouts. Advanced control incorporates adaptive signal technologies adjusting timings based on real-time conditions (Stevanovic et al., 2010).

📡 ii. Traffic Monitoring and Detection

Continuous acquisition of traffic data using sensing technologies. Modern systems employ CCTV cameras, radar sensors, GPS probe data, and computer vision algorithms. Deep learning detectors like YOLO and Faster R-CNN have significantly improved accuracy (Redmon et al., 2016; Ren et al., 2015).

⚠ iii. Incident and Demand Management

Rapid detection and coordinated response to traffic-disruptive events. Demand management focuses on reducing peak-period congestion through staggered work hours, enhanced public transport, and congestion pricing (FHWA, 2013; Litman, 2017).

📡 iv. Information and Communication Systems

Disseminating real-time traffic information via roadside message boards, mobile apps, radio broadcasts, and control-room interfaces. Modern ITS incorporates big data analytics and ML for traffic forecasting (Li et al., 2021).

⚖ v. Policy and Regulatory Framework

Specifies acceptable road-user behavior, vehicle standards, enforcement procedures, and urban mobility strategies. Effective enforcement mechanisms are essential for ensuring compliance (WHO, 2018; OECD/ITF, 2019).

2.3 Deep Learning for Traffic Detection

Deep learning has emerged as a foundational technology in ITS. Unlike traditional ML that relies on hand-crafted features (HOG, SIFT), deep learning models learn hierarchical feature representations directly from raw image data (LeCun et al., 2015). This enables CNNs to handle occlusion, scale variation, shadows, and variable weather conditions.

Vehicle Detection

Deep learning models reliably identify multiple vehicle types using learned semantic features (Ren et al., 2015).

Multi-Object Tracking

Combined with SORT, DeepSORT, ByteTrack for consistent vehicle tracking across frames (Zhang et al., 2021).

Traffic Volume Estimation

Automated computation of vehicle counts, speeds, and flow patterns in real-time.

Real-Time Performance

Lightweight backbones, feature-fusion, and quantization enable edge-device deployment (Bochkovskiy et al., 2020).

3.9 Evaluation Metrics

Precision (P)

TP / (TP + FP) — Proportion of correct positive predictions.

Recall (R)

TP / (TP + FN) — Ability to detect all relevant objects.

F1-Score

2 × (P × R) / (P + R) — Harmonic mean of precision and recall.

IoU

Intersection / Union — Overlap between predicted and ground truth boxes.

mAP

Mean Average Precision across all classes and IoU thresholds.

📈 Chapter Four

Results and Discussion

4.1 Introduction

This chapter presents and discusses the experimental results from the development and evaluation of the proposed deep learning–based traffic vehicle detection and classification system. Emphasis is placed on the effectiveness of the YOLO and Faster R-CNN frameworks in achieving real-time performance while maintaining high detection accuracy.

4.2 Model Development and Implementation

The development followed a structured workflow: environment configuration, dataset integration, model initialization, training, validation, and performance evaluation. A cloud-based computational environment (Google Colab) with GPU acceleration was adopted.

YOLOv11 Implementation

The YOLO-based detection framework was implemented as a single-stage model optimized for real-time performance. Pretrained weights enabled transfer learning. Training was completed in approximately 1.012 hours (~60 minutes) over 50 epochs, achieving an ultimate inference speed of 11.9ms (approx 84 FPS) on the local RTX 3070.

Faster R-CNN Implementation

Implemented using the PyTorch framework with CUDA-enabled libraries for GPU acceleration. A custom VehicleDataset class bridged the raw dataset directly to the PyTorch training engine using a custom collate function. The model utilized transfer learning from COCO-pretrained ResNet-50 backbone networks. Training 10 epochs natively reached a plateau with an average bounding box/classification loss converging around 0.1494. Final weights were exported to a 164MB .pth tensor format.

Key Finding: YOLO outperforms Faster R-CNN explicitly for Real-Time Deployment. While Faster R-CNN's two-stage Region Proposal Network provides strong localization at the cost of high VRAM (164MB weight size) and slower inference (15 FPS), YOLOv11 processed images at a blazing 84 FPS while consuming only a fraction of storage (5.5MB). For edge-device deployment, YOLOv11 is the optimal choice.

4.8 Conclusion

This project successfully developed a YOLOv11-based deep learning system for real-time vehicle detection, classification, and traffic volume estimation. By combining public and custom datasets, the system accurately identified multiple vehicle categories — two-wheelers, three-wheelers, four-wheelers, and large vehicles. YOLOv11 provides high detection accuracy, efficient processing speed, and adaptability for deployment in resource-constrained urban environments. The system offers a practical and scalable solution for improving traffic monitoring and management.

Dataset Engineering & Pre-Processing

Data Sanitization Architecture

Before deep learning models can train, raw datasets from mixed Kaggle sources must be scrubbed, formatted into YOLO arrays, and correctly mapped to prevent collision. The dataset pipeline (executed via arrange_dataset.py) isolates this workflow to prevent data corruption during repetitive testing, utilizing three targeted paradigms:

  • Standard Indexing (Four/Two Wheeler): Utilizes direct VOC-to-YOLO bounding box normalization and sequential prefix generation.
  • Override Convergence (Large Vehicle): Since heavy vehicles were disjointed (labeled individually as 'tractor' and 'truck'), stringent keyword filters are bypassed. The script mechanically forces all geometries into the unifying class_id = 1 constraint. Furthermore, decimal floating-point coordinates are dynamically sanitized to prevent structural exceptions during tensor mapping.
  • Post-Split Remapping (Three Wheeler): Tricycle imagery arrived with colliding predefined IDs (class_id = 0). An automated sanitization sweep reads every file globally and forces the mapping to class_id = 2 directly prior to dataset compilation.

Troubleshooting

⚠ CUDA Not Detected After Install

If torch.cuda.is_available() returns False after installing CUDA PyTorch, force reinstall:

pip install --force-reinstall torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu128

Ensure your NVIDIA driver version ≥ 581. Check with nvidia-smi.

⚠ CUDA Out of Memory

Reduce batch size to 8 or 4 in training config. Alternatively, set FORCE_CPU=1 environment variable to run entirely on CPU.

⚠ Missing Label Files

Run python fix_all_labels.py to convert XML annotations to YOLO format, remap class IDs, and validate all label files across all splits.

⚠ Low Detection Accuracy

Increase training epochs (50+), verify dataset annotations, and ensure class balance. Check the label distribution with fix_all_labels.py validation step — target: 4 classes with reasonable annotation counts.

⚠ Flask App Not Starting

Ensure all dependencies are installed: pip install flask ultralytics opencv-python. Check that the trained weights file exists at the expected path in config.
